# Security Policy

## Supported Versions

Security updates are provided for the latest development line.

| Version | Supported |
| ------- | --------- |
| 0.x     | Yes       |
| < 0.0.0 | No        |

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Report vulnerabilities by contacting the maintainer directly with:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested mitigation (if known)

For this project, contact:

- GitHub: @adnanian

## What to Expect

- Acknowledgement within 3 business days.
- Initial triage and impact assessment.
- Fix timeline based on severity and complexity.
- Public disclosure after a fix is available.
